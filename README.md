# Conceito de Log: no contexto da tecnologia, um Log (ou registro) é um arquivo de texto gerado automaticamente por um software, sistema operacional, servidor ou dispositivo de rede que documenta eventos cronológicos. Pense no log como a "caixa preta" de um avião ou o "diário de bordo" de um navio: ele grava tudo o que acontece, exatamente quando acontece, para que se possa entender o comportamento do sistema posteriormente.
# 1. A Estrutura de um Log:
#Embora o formato varie dependendo do sistema, a maioria das entradas de log contém três elementos essenciais:
#Timestamp (Carimbo de Data/Hora): O momento exato (data, hora, minuto, segundo e milissegundo) em que o evento ocorreu.
#Nível de Severidade: Classifica a importância do evento (ex: INFO para informação geral, WARN para avisos, ERROR para falhas, FATAL para erros críticos).
#A Mensagem: A descrição do que aconteceu (ex: "Usuário X fez login", "Falha de conexão com o banco de dados").
#2. Para que servem os Logs?
Os logs são fundamentais para três pilares principais da TI:
#Troubleshooting (Resolução de Problemas): Quando um site cai ou um aplicativo trava, o desenvolvedor consulta os logs para ver qual foi o último evento registrado antes da falha. É a principal ferramenta de "detetive" para corrigir bugs.
#Segurança e Auditoria: Logs registram quem acessou o sistema, de onde (endereço IP) e quais arquivos foram modificados. Isso é vital para investigar invasões ou vazamentos de dados.
#Monitoramento de Desempenho: Analisar logs pode revelar se um sistema está ficando lento em horários específicos ou se está consumindo muita memória.
